# Acceptance Criteria

1) Gateway Service API
   - Type: integration
   - Evidence: Run `grpcurl` against `localhost:50051`.
   ```bash
   grpcurl -plaintext -d '{}' localhost:50051 ucl.gateway.v1.GatewayService/ListActions
   ```
   - Expect: Response contains `input_schema_json` populated with actual JSON (not empty/null) for actions like `jira.create_issue`.

2) Connector Coverage
   - Type: code/review
   - Evidence: Verify `RegisterActionSchema` calls exist in:
     - `internal/connector/jira` (Done)
     - `internal/connector/github`
     - `internal/connector/confluence`
     - `internal/connector/minio`
     - `internal/connector/jdbc`

3) Agent Tool Discovery
   - Type: e2e
   - Evidence: Query Agent's `/tools` endpoint (via local curl or mcp-inspector).
   - Expect: Tool definitions include `inputSchema` matching the registered definitions.

4) LLM Function Calling (Optional/Manual)
   - Type: manual
   - Evidence: Agent successfully invokes a tool with parameters generated by the LLM (e.g., "Create a jira ticket...").
